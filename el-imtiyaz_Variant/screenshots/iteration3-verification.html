Issue 5.2 MEDIUM

QuoteBlock 'advances' was an invention — replaced with 'remboursement'

BeforeThe QuoteBlock entity had an 'advances' field that doesn't exist in Excel. netPayable was computed as `subTotal - advances - discounts`. Excel's actual Devis formulas are: I31: =I27 - I29 (subtotal − réduction) I31: =I27 - I29 - I30 (subtotal − réduction − remboursement) where I29 is the discount and I30 is a 'remboursement' credit.
AfterNew `remboursement` field added to QuoteBlock (migration 006). compute() now returns: netPayable = max(0, subTotal - discounts - remboursement) The legacy `advances` field is kept for backward compat but no longer used in the formula. Repository.create() now honours the computed subTotal / netPayable / schoolFeeTax instead of hardcoding them to 0.

Issue 5.3/5.4/9.2 HIGH

5% 'tax' was actually a conditional early-payment bonus (D35 note)

BeforeschoolFeeTax was persisted as SUM(fraisScolaire) * 0.05 UNCONDITIONALLY — a static computed field. Excel's D35 note says: "Nb 01: une remise de 5% sois [amount] est rajoutée si le paiement est effectué en totalité avant le 30 juin" It's a CONDITIONAL bonus on early payment, not a tax. The 5% figure does NOT appear in the Montant Total DZD cell (I31).
AfterNew `paymentDate` column on QuoteBlock. The bonus is now computed only when `paymentDate <= ${year}-06-30` (year-agnostic cutoff). When the condition is not met (or paymentDate is missing), schoolFeeTax is persisted as 0. qualifiesForEarlyPaymentBonus() is exported for direct testing. The field is also no longer a 'tax' — it's an informational discount.

Issue 8.7 LOW

Duplicate devis numbers now surface an advisory warning

BeforeExcel's Devis sheet contains duplicate block identifiers (e.g. two blocks share '0103/2021/2022', two share '0104/2021/2022', two share '0107/2021/2022'). The software's quote_blocks table had no uniqueness constraint on `name` AND no detection logic. An operator could create any number of duplicate-named blocks with no feedback at all.
AfterQuoteService.checkDuplicateName() scans existing non-deleted quote blocks and returns a warning when a block with the same name already exists. The warning is surfaced via the same validateInput() pipeline as the Nb 02 rule. The save is NOT blocked — Excel allows duplicates (they may be intentional re-quotes), but the operator gets clear feedback to verify.
