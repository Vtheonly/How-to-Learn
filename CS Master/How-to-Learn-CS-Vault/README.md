# How to Learn Computer Science — Obsidian Vault

An evidence-based learning system for mastering computer science, grounded in cognitive science, educational psychology, and expertise research.

## Quick Start

1. Download and install [Obsidian](https://obsidian.md) (free).
2. Open Obsidian → "Open folder as vault" → select this folder.
3. Read [`00-Index/Start-Here.md`](00-Index/Start-Here/Start-Here.md) for onboarding.
4. Follow the setup guide in [`11-Setup-and-Plugins/Obsidian-Setup-Guide.md`](11-Setup-and-Plugins/Obsidian-Setup-Guide.md) to install recommended plugins.
5. Open the visual vault map: [`00-Index/Vault-Map.canvas`](00-Index/Vault-Map.canvas).

## What's in the Vault

- **80 markdown notes** organized in 13 thematic folders
- **5 core pillars** of cognitive-science-grounded learning (Foundations, Reading, Triage, Load, Schema)
- **10 CS domain maps** showing how to attack each subfield (math, algorithms, OS, networks, databases, compilers, distributed systems, ML, PL)
- **7 Obsidian templates** (daily log, triage card, schema map, SR queue, project log, concept note, reading note)
- **4-phase roadmap** translating the cognitive science into a 3-7 year arc
- **A visual canvas** showing the full learning architecture
- **A complete bibliography** of peer-reviewed sources (Sweller, Ericsson, Kintsch, Gentner, Rumelhart, Spiro, Meyer-Land, Roediger-Karpicke, Ebbinghaus, and many CS pioneers)

## Source Standards

Every recommendation is grounded in:
- Peer-reviewed literature in cognitive psychology, educational psychology, learning sciences, and expert-performance research
- Established theoretical frameworks: Schema Theory (Rumelhart/Anderson), Structure Mapping Theory (Gentner), Cognitive Load Theory (Sweller), Long-Term Working Memory (Ericsson & Kintsch), Cognitive Flexibility Theory (Spiro), Threshold Concepts (Meyer & Land), Testing Effect (Roediger-Karpicke)
- Empirical/historical case studies of Knuth, Dijkstra, McCarthy, Bell Labs engineers, Norvig, and modern practitioners (Karpathy, tptacek, Victor)

See `99-References/Bibliography.md` for the full source list.
