# CS-Mastery-Vault

> A research-grounded Obsidian vault that teaches you **how to learn computer science** — not computer science itself.
>
> Built from peer-reviewed cognitive science, educational psychology, expertise research, and learning sciences.

---

## What this vault is

This vault is a **practical learning system**. It transforms decades of research on how experts acquire massive technical knowledge into a concrete, day-by-day protocol you can apply to your study of computer science, software engineering, and AI.

It does **not** teach you what a B-tree is. It teaches you:

- **How to learn** a B-tree so you actually retain it.
- **How to connect** the B-tree to filesystems, ASTs, and Merkle trees so the schema transfers.
- **How to triage** when to learn it deeply vs. skim vs. defer.
- **How to retrieve, space, interleave, and test** the knowledge so it sticks.
- **How to organize your Obsidian vault** so the structure itself accelerates learning.

## The single most important idea

> The fastest learners in computer science did **not** study every topic independently. They built a library of **reusable cognitive schemas** (state, graph, optimization, concurrency, information flow, …) and then mapped each new technology, language, and system onto those schemas.
>
> Mastery is not about coverage. It is about **schema density** and **retrieval strength**.

Everything in this vault is a mechanism for building schema density and retrieval strength.

## The four vault rules

1. **Every note connects to a schema.** New concept? Find the schema it instantiates and link to it.
2. **Every concept ends in retrieval.** No note is "done" until it has at least one self-test question.
3. **Every session is logged.** Use the daily template — even a one-line log is enough.
4. **Every week, abstract.** Weekly review = identify patterns, merge notes, create schema links.

## Design principles

- **Evidence first.** Every recommendation cites the underlying research. No productivity bro-science.
- **Actionable, not abstract.** Every principle is paired with a protocol or template.
- **Schema-centric.** The vault is organized so that *organizing the vault* is itself schema-building practice.
- **Mastery-typed.** Every concept is tagged with its current mastery level (1–6) using the rubric in `05_Roadmap/R3 — Mastery Rubric`.
