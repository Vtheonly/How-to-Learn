## The non-negotiable four

If you only adopt four methods, adopt these (highest effect sizes, broadest applicability):

1. **[[03_Methods/M1 — Retrieval Practice|M1 Retrieval Practice]]** — every session.
2. **[[03_Methods/M2 — Spaced Repetition|M2 Spaced Repetition]]** — every day, 15 min.
3. **[[03_Methods/M4 — Worked Examples|M4 Worked Examples]]** — every new topic, *before* problem-solving.
4. **[[03_Methods/M10 — Strategic Triage|M10 Strategic Triage]]** — every week.

The other six amplify these four; they do not replace them.

---

## Common misuses to avoid

| Method | Common misuse | Right use |
|--------|---------------|-----------|
| Retrieval practice | Flashcards for definitions only | Multi-level prompts (recall → explain → derive → implement → diagnose → transfer) |
| Spaced repetition | Reviewing 500 cards/day | 15–30 quality cards/day, deeper prompts |
| Interleaving | Random shuffling of unrelated topics | Interleave within a schema family (e.g., 3 concurrency problems) |
| Worked examples | Reading passively | Cover solution, attempt, then compare; self-explain each step |
| Self-explanation | Explaining to a friend once | Writing it down; revising it after a delay |
| Concept mapping | Drawing pretty graphs | Forcing yourself to label the **edges** with the relation type |

## How to plan a study session

Use the daily template (`06_Templates/Daily Session.md`) which embeds this pattern:

```
1. Triage   (M10)         — 5 min
2. Worked example (M4)    — 15 min
3. Generative production (M8)  — 30 min
4. Self-explanation (M5)  — 10 min
5. Retrieval (M1)         — 10 min
6. Log + retrieval prompts (M1) — 5 min
```

For a 90-minute session, scale proportionally. The non-negotiable parts are **(2), (3), (5)** — they are where the actual learning happens.
